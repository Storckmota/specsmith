## What This Confirms

1. **Qwen models work with SpecSmith out of the box.** The existing `PROVIDER=api` path is
   fully Qwen-compatible — no code changes were needed.
2. **OpenRouter is a valid Qwen delivery mechanism.** The `/chat/completions` endpoint at
   `https://openrouter.ai/api/v1` is OpenAI-compatible and accepted by the `ApiProvider` class.
3. **All 5 agents produce schema-valid output.** Zod validation passed for every stage of the
   pipeline.
4. **No false AMD/vLLM claims are made.** The `providerMode` field in the response correctly
   returns `"API mode"` when `PROVIDER=api`, not `"AMD/Qwen mode"`.

## AMD / vLLM Status

AMD Developer Cloud / vLLM mode (`PROVIDER=amd`) is **planned but not yet active**.

Activation requires AMD Developer Cloud GPU credits and a running vLLM instance.

Qwen models are the intended workload for AMD mode — today's validation confirms the model
produces correct output through the pipeline. The only remaining step is wiring the AMD endpoint.
